# Notes — 작성 원칙

개념 노트는 문제를 풀면서 등장하는 개념을 정리하는 공간이다.
단순한 정의 나열이 아니라, 개념 간의 관계와 실무 맥락을 중심으로 작성한다.

---

## 폴더 구조

- 상위 개념이 존재하는데 같은 레벨에 두지 않는다
- 같은 레벨의 개념을 근거 없이 서로 다른 폴더에 분리하지 않는다
- 폴더명을 임의로 만들지 않는다 — 개념의 계층 구조에 없는 이름을 쓰지 않는다

---

## 각 파일에 반드시 담을 것

- 이 개념의 **상위 개념**이 무엇인가
- **비슷하거나 혼동되는 개념**과의 차이
- **관련 문제 번호** (Q번호)
- 실무에서 **어떤 맥락**으로 등장하는가

---

## 작성 스타일

- 비교는 **테이블**로
- 코드는 **코드블록**으로
- 교과서식 설명 금지 — 짧고 맥락 중심으로
- 관련 개념 파일이 있으면 **링크**로 연결

---

## 파일 네이밍

- 소문자 + 언더스코어: `concept_drift.md`
- 파일명만 봐도 내용을 알 수 있게
