# Concept Notes — 작성 원칙

개념 노트는 문제를 풀면서 등장하는 개념을 정리하는 공간이다.
단순한 정의 나열이 아니라, 개념 간의 관계와 실무 맥락을 중심으로 작성한다.

---

## 파일 구성

- 개념은 주제별 폴더로 묶되, 폴더 구조는 유동적으로 확장 가능하다
- 하나의 개념이 여러 주제에 걸치면 가장 핵심 주제 폴더에 위치시킨다
- 개념이 세분화되어 단일 파일로 감당이 안 될 경우, 하위 폴더를 추가로 생성하고 tree 구조를 확장해도 된다

---

## 각 파일에 반드시 담을 것

- 이 개념의 **상위 개념**이 무엇인가
- **비슷하거나 혼동되는 개념**과의 차이
- **관련 문제 번호** (Q번호)
- 실무에서 **어떤 맥락**으로 등장하는가

---

## 작성 스타일

- 비교는 **테이블**로
- 코드는 **코드블록**으로
- 짧고 명확하게, 교과서식 설명 금지
- 관련 개념 파일이 있으면 **링크**로 연결
